## UX/UI: Административный раздел «Кнопки БП»

Здесь описывается интерфейс реестра и настройки логики:

- размещение в меню: «Настройки → Мои модули → Кнопки БП»;
- макет списка полей: колонки, фильтры, сортировка;
- формы редактирования: какие поля доступны и как они валидируются;
- требования к адаптивности и совместимости с темами Bitrix24.

---

### 1. Размещение в меню и точка входа

- Путь: **«Настройки → Мои модули → Кнопки БП»**.
- Пункт меню:
  - название: «Кнопки БП»;
  - подпись/описание: «Управление пользовательскими кнопками в карточках CRM».
- При нажатии открывается страница с реестром настроек (список записей `my_bpbutton_settings`).

---

### 2. Макет списка (реестр полей)

#### 2.1. Основные колонки

Рекомендуемый набор колонок для списка:

- **ID** — внутренний идентификатор записи настроек.
- **Поле (FIELD_ID / код)**:
  - отображается человеко‑читаемое название пользовательского поля и технический код (например, `UF_BPBTN_DEAL_RUN`);
  - по возможности — ссылка на стандартный интерфейс управления этим полем.
- **Сущность (ENTITY_ID)**:
  - тип CRM‑сущности (Лид, Сделка, Контакт, Компания, Смарт‑процесс);
  - может отображаться в локализованном виде («Сделка», «Лид» и т.п.).
- **URL обработчика (HANDLER_URL)**:
  - укороченный вид (с подсказкой/tooltip при наведении);
  - удобен для визуальной проверки настроек.
- **Заголовок окна (TITLE)**:
  - текст, который увидит пользователь в шапке SidePanel.
- **Ширина (WIDTH)**:
  - строка вида `70%` или `1200`;
  - может сопровождаться подсказкой об интерпретации (px/%).
- **Активность (ACTIVE)**:
  - чекбокс/значок «Вкл/Выкл»;
  - позволяет быстро отключить кнопку без удаления поля.
- **Дата изменения (UPDATED_AT)**:
  - для аудита, кто и когда менял настройки (при наличии расширенной информации).

#### 2.2. Фильтры и сортировка

Фильтры по основным сценариям администратора:

- по сущности (`ENTITY_ID`): показать только кнопки для Сделок/Лидов и т.п.;
- по активности (`ACTIVE = Y/N`);
- по части названия поля или техническому коду;
- по части URL обработчика (полезно при массовой смене endpoint‑ов).

Сортировка:

- по ID (по умолчанию, по убыванию);
- по названию поля;
- по дате изменения (`UPDATED_AT`).

---

### 3. Действия администратора в реестре

Типовые действия в списке:

- **Открыть настройки кнопки**:
  - клик по названию поля или иконке «Редактировать»;
  - открывается форма редактирования на отдельной странице или в SidePanel.
- **Быстрое включение/выключение**:
  - изменение флага активности (`ACTIVE`) прямо из списка (inline‑редактирование или массовое действие).
- **Массовые операции** (опционально):
  - активировать/деактивировать выбранные записи;
  - удалить настройки (при этом следует явно предупредить о последствиях).

Важный UX‑момент:

- любые destructive‑действия (удаление настроек) должны сопровождаться подтверждением.

---

### 4. Форма редактирования настроек

Форма редактирования должна быть простой и понятной:

**Поля формы:**

- `FIELD_ID` / Название поля:
  - только для чтения (read‑only);
  - отображает человеко‑читаемое имя и технический код.
- `ENTITY_ID`:
  - только для чтения или выбирается из списка (если поддерживаются ограничения по сущностям).
- `HANDLER_URL`:
  - текстовое поле с проверкой формата URL (может быть относительным или абсолютным);
  - подсказка, какой обработчик ожидается (внутренний скрипт, внешний сервис и т.п.).
- `TITLE`:
  - строка заголовка SidePanel;
  - желательна валидация по длине.
- `WIDTH`:
  - поле для ширины;
  - подсказка о допустимых значениях (число — пиксели, строка с `%` — проценты).
- `ACTIVE`:
  - чекбокс включения/выключения.

**Валидация и UX:**

- кнопка сохранения активна только при корректно заполненных обязательных полях;
- при ошибке валидации:
  - подсветка полей с ошибками;
  - краткое пояснение рядом с полем;
  - фокус переводится на первое проблемное поле.

**Уведомления:**

- при успешном сохранении:
  - ненавязчивое уведомление в стиле Bitrix24 («Настройки кнопки сохранены»);
- при ошибке:
  - понятное сообщение без технических деталей (подробнее — в логах).

---

### 5. Адаптивность и темы Bitrix24

- Интерфейс должен корректно выглядеть:
  - в светлой и тёмной темах Bitrix24;
  - при изменении ширины экрана (особенно если реестр/форма открываются в SidePanel).
- Использовать стандартные компоненты и классы Bitrix24 для:
  - таблиц, фильтров, форм;
  - кнопок действий (`ui-btn`, `ui-btn-primary`, `ui-btn-link` и т.п.).
- Избегать жёстких inline‑стилей и нестандартных цветов, которые могут конфликтовать с темами.
