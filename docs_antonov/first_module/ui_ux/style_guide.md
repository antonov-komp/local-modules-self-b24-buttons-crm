## UI Style Guide для модуля «BP Button Field»

Документ задаёт минимальный UI‑гайд для всех интерфейсов модуля:
- кнопка в карточке CRM;
- админ‑реестр и формы настроек;
- вспомогательные экраны (SidePanel, если используются).

Цель — обеспечить единообразный внешний вид в духе Bitrix24 и избежать «самодельного» UI.

---

### 1. Общие принципы

- **Следуем Bitrix24 UI Kit.**
  - Используем готовые компоненты и классы (`ui-btn`, `ui-alert`, `ui-typography` и т.п.), где это возможно.
  - Не изобретаем собственные визуальные паттерны без необходимости.

- **Минимум кастомных стилей.**
  - Собственные CSS‑классы вводим только при недостаточности стандартных.
  - Новые классы оформляем в `/local/css/my.bpbutton/*.css` и не смешиваем с inline‑стилями.

- **Уважение тем (светлая/тёмная).**
  - Не задаём жёсткие цвета фонов и текста, противоречащие темам Bitrix24.
  - Оперируем нейтральными оттенками и полупрозрачными цветами, если нужно подчеркнуть элементы.

---

### 2. Кнопки (карточка CRM и админка)

#### 2.1. Базовые классы

- Обязательные:
  - `ui-btn` — общий базовый стиль кнопки.

- Основные варианты:
  - `ui-btn-primary` — ключевые действия (нажатие кнопки в карточке CRM, сохранение настроек).
  - `ui-btn-light-border` — второстепенные действия (отмена, дополнительные опции).
  - `ui-btn-link` — действия в «ссылочном» стиле (посмотреть детали, перейти и т.п.).

- Состояния:
  - `ui-btn-wait` — состояние загрузки.
  - `ui-btn-disabled` — выключенное состояние.

#### 2.2. Допустимые модификаторы

- `ui-btn-round` — при необходимости более мягких углов (по вкусу, но без чрезмерного «pill»‑эффекта).
- `ui-btn-icon-*` — при добавлении иконок (строго из набора Bitrix).

#### 2.3. Запрещено

- Жёсткая перезапись ключевых свойств:
  - `background-color`, `color` с произвольными значениями, конфликтующими с темами Bitrix24.
  - собственные тени/анимации, радикально отличающиеся от остального UI.

- Использование нестандартных HTML‑элементов вместо кнопок (`div` без роли, `span` без семантики), кроме случаев, когда этого явно требует Bitrix.

---

### 3. Типографика и шрифты

- Используем системные настройки Bitrix24:
  - не подключаем свои шрифты;
  - не переопределяем глобально семейства шрифтов.

- Для текстов:
  - заголовки страниц/блоков — классы из UI Kit (`ui-typography-heading-*`, если используются) или стандартные `<h1>–<h3>`.
  - подзаголовки и описания — обычный текст, максимум лёгкое выделение (`<strong>`, `<em>`).

- Запрещено:
  - жёстко задавать `font-family` вне очень локальных случаев (например, декоративный элемент внутри SidePanel).

---

### 4. Отступы и сетка

- Следуем типичным для Bitrix24 отступам:
  - вертикальные отступы между блоками: 16–24px;
  - горизонтальные отступы внутри элементов: 8–16px.

- В таблицах/формах:
  - используем стандартные admin‑компоненты Bitrix (admin‑таблицы, формы);
  - минимизируем ручные `margin/padding`.

- Для SidePanel:
  - контент выравнивается по стандартным отступам Bitrix (обычно 24px по краям основного контейнера).

---

### 5. Адаптивность

- Реестр и формы:
  - должны корректно отображаться в стандартной верстке админки Bitrix (в том числе при сужении окна/SidePanel).
  - таблица может иметь горизонтальную прокрутку при большом числе колонок, но основные поля (поле/сущность/активность) должны оставаться читаемыми.

- Кнопка в карточке CRM:
  - не должна «ломать» строку больше, чем это делает стандартный UI;
  - текст кнопки должен быть достаточно коротким, чтобы укладываться в одну-две строки.

---

### 6. Цвета и состояния ошибок

- Для ошибок и предупреждений:
  - по возможности используем стандартные компоненты:
    - `BX.UI.Notification.Center` с типичным оформлением;
    - `ui-alert ui-alert-danger` / `ui-alert-warning` (если нужны статические блоки).

- Не задаём собственные «красные/желтые» цвета:
  - используем переменные/классы Bitrix, чтобы соответствовать темам.

---

### 7. Чек‑лист UI‑ревью

1. **Кнопки**
   - [ ] Все кнопки используют `ui-btn` и один из допустимых модификаторов (`ui-btn-primary`, `ui-btn-light-border`, `ui-btn-link`).
   - [ ] Нет нестандартных inline‑стилей, перекрашивающих кнопки в чуждые цвета.

2. **Типографика**
   - [ ] Используются стандартные шрифты Bitrix24, нет глобальных переопределений `font-family`.
   - [ ] Заголовки и текст читаемы и не конфликтуют со стандартным UI.

3. **Темы и адаптивность**
   - [ ] В светлой и тёмной темах элементы выглядят корректно.
   - [ ] На узких экранах (SidePanel, уменьшенное окно) интерфейс не разваливается.

4. **Стилизация ошибок**
   - [ ] Для ошибок и предупреждений используются стандартные UI‑паттерны Bitrix (уведомления, алерты).
