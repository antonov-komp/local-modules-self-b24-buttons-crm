## UX/UI: Кнопка в карточке CRM и смарт-процессов

Документ описывает пользовательский опыт и внешний вид:

- Как выглядит поле типа `bp_button_field` в карточке (состояния, стили).
- Поведение при клике: лоадер, блокировка повторных нажатий.
- Открытие слайдера (`BX.SidePanel`) и ожидания по скорости/отзывчивости.
- Соответствие Bitrix24 UI Kit (например, `ui-btn-primary`, `ui-btn-light-border`).

---

### 1. Состояния кнопки

Кнопка на базе `bp_button_field` должна поддерживать следующие состояния:

- **idle (по умолчанию)**  
  - Кнопка активна, отображается основной текст (из названия поля или настроек).
  - Курсор: `pointer`.
  - Нет индикаторов загрузки.

- **loading**  
  - Активируется сразу после клика пользователя.
  - Кнопка визуально переходит в состояние ожидания:
    - добавляется модификатор `ui-btn-wait` (или аналогичный класс Bitrix24);
    - повторное нажатие недоступно до завершения операции.

- **disabled**  
  - Используется, если по бизнес‑логике кнопка временно недоступна
    (например, нет настроек или пользователь не имеет прав).
  - Стиль: приглушённый (например, `ui-btn-disabled`), курсор `default`.
  - Кнопка не реагирует на клики.

- **error (визуальный отклик)**  
  - При ошибке загрузки конфигурации или открытии SidePanel
    кнопка может:
    - кратко подсветиться (красная рамка/фон) или
    - вернуться в состояние `idle`, но вместе с уведомлением (см. раздел об ошибках).

Переходы между состояниями:

1. `idle` → `loading` — по первому клику.
2. `loading` → `idle` — по успешному получению конфигурации и запуску SidePanel.
3. `loading` → `error`/`idle` — при ошибке (с отображением уведомления).
4. `any` → `disabled` — при изменении бизнес‑условий или настроек (например, поле деактивировано).

---

### 2. Визуальные стили (Bitrix24 UI Kit)

Базой для кнопки служат классы Bitrix24 UI Kit:

- обязательные классы:
  - `ui-btn` — базовый стиль кнопки;
  - один из вариантов оформления:
    - `ui-btn-primary` — акцентная кнопка (рекомендуется для ключевых действий);
    - `ui-btn-light-border` — второстепенная, лёгкий контур;
    - `ui-btn-link` — кнопка‑ссылка (для менее критичных действий).
- дополнительные модификаторы:
  - `ui-btn-round` (по необходимости, если требуется скругление);
  - `ui-btn-icon-*` — при добавлении иконок;
  - `ui-btn-wait` — состояние загрузки.

Рекомендации:

- по умолчанию использовать `ui-btn ui-btn-primary`;
- позволить администратору в будущем выбирать стиль из ограниченного набора
  (см. настройки визуала в `local_button.md`);
- не использовать нестандартные цвета, конфликтующие с темами Bitrix24.

### 3. Поведение при клике и ожидания по отзывчивости

- Время реакции:
  - визуальный отклик (переход в `loading`) должен происходить **мгновенно** (до 100 мс);
  - открытие SidePanel — в пределах 1–2 секунд при нормальном соединении.
- UX‑паттерн:
  1. Пользователь нажимает кнопку.
  2. Кнопка переходит в состояние `loading` (`ui-btn-wait`, блокировка повторного клика).
  3. Отправляется запрос к `ButtonController.getConfigAction`.
  4. При успехе:
     - кнопка возвращается в `idle`;
     - открывается `BX.SidePanel` с переданными параметрами.
  5. При ошибке:
     - кнопка возвращается в `idle` (или кратко подсвечивается ошибкой);
     - пользователю показывается уведомление (см. ниже).

---

### 4. UX‑паттерны при ошибках

Типовые ошибки:

- нет прав на элемент CRM;
- отсутствуют настройки кнопки (`my_bpbutton_settings`);
- кнопка деактивирована (`ACTIVE = 'N'`);
- внутренняя ошибка/недоступен обработчик.

Поведение:

- кнопка **не должна зависать** в состоянии загрузки;
- пользователь должен получить **краткое, понятное сообщение**:
  - без технических деталей;
  - с указанием, что именно пошло не так (нет прав / не настроено / временная ошибка).

Рекомендуемый механизм уведомлений:

- через стандартный UI Bitrix24:
  - `BX.UI.Notification.Center.notify({ content: '...', autoHideDelay: 5000 })`;
- текст сообщений должен быть локализован (`Loc::getMessage`) и соответствовать:
  - кодам ошибок `INVALID_SESSION`, `ACCESS_DENIED`, `SETTINGS_NOT_FOUND`, `BUTTON_INACTIVE`, `INTERNAL_ERROR`
    (см. `api/ajax_controller.md`).

Примеры сообщений:

- при `ACCESS_DENIED`: «Недостаточно прав для выполнения действия.»
- при `SETTINGS_NOT_FOUND`: «Кнопка не настроена. Обратитесь к администратору.»
- при `INTERNAL_ERROR`: «Произошла ошибка. Попробуйте позже или обратитесь к администратору.»
