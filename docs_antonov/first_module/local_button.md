Разработка модуля «BP Button Field» для 1С-Битрикс24 (Коробка)
1. Цель проекта
Создание программного модуля, расширяющего стандартные возможности CRM Битрикс24 путем добавления нового типа пользовательского поля — «Кнопка управления логикой». Модуль должен позволять администратору создавать функциональные кнопки в карточках CRM, которые при нажатии открывают интерфейсы (Iframe/SidePanel) или запускают бизнес-логику без перезагрузки страницы.

2. Функциональные требования
2.1. Регистрация кастомного типа поля (User Type)
Идентификатор типа: bp_button_field.
Доступность: Поле должно быть доступно для создания во всех сущностях CRM (Лиды, Сделки, Контакты, Компании) и Смарт-процессах через стандартный интерфейс «Добавить поле».
Параметры поля (при создании):
Название: Заголовок поля в системе.
Технический ID: Уникальный код кнопки для идентификации в коде модуля.
Визуал: Выбор стиля кнопки (на базе Bitrix UI-Kit, например: ui-btn-primary, ui-btn-light-border).
2.2. Поведение в интерфейсе CRM (View & Action)
Отображение: В режиме просмотра карточки CRM вместо текстового значения выводится активная кнопка.
Механика клика: При нажатии кнопка переходит в состояние state-wait (лоадер), предотвращая повторные нажатия.
Действие: Вызов JS-расширения модуля, которое открывает BX.SidePanel.Instance (слайдер) с загрузкой контента через Iframe.
Контекст: В Iframe/обработчик должны передаваться параметры: ENTITY_ID (тип сущности), ELEMENT_ID (ID записи), FIELD_ID (ID конкретного поля) и USER_ID (кто нажал).
2.3. Административный реестр полей (Конструктор)
В административном разделе Битрикса («Настройки» -> «Мои модули» -> «Кнопки БП») должен быть реализован реестр:
Авто-регистрация: При создании в CRM поля типа bp_button_field, оно автоматически появляется в списке модуля.
Управление логикой: Для каждого поля в реестре администратор может задать:
URL обработчика: Ссылка на файл или внешний сервис, который откроется в Iframe.
Заголовок окна: Текст, отображаемый в шапке слайдера.
Ширина окна: Настройка ширины SidePanel (в px или %).

3. Техническая архитектура (Backend D7)
3.1. Структура хранения данных
Для управления логикой кнопок используется собственная ORM-сущность.
Таблица: my_bpbutton_settings.
Класс: My\BpButton\Internals\SettingsTable (наследник DataManager).
Связи: Поле FIELD_ID связывает запись с системной таблицей b_user_field.
3.2. Обработка событий (Events)
main:OnUserTypeBuildList — регистрация типа поля в системе при инициализации модуля.
main:OnAfterUserFieldAdd — автоматическое создание записи в SettingsTable при добавлении нового поля в CRM.
main:OnUserFieldDelete — каскадная очистка настроек модуля при удалении поля.
3.3. AJAX-контроллер
Реализация собственного контроллера на базе Bitrix\Main\Engine\Controller для безопасной обработки запросов от кнопок, проверки прав доступа пользователя и возврата конфигурации SidePanel.

4. Стек технологий и Стандарты
Язык: PHP 8.x (стандарты D7).
Frontend: Bitrix Core JS (BX.Main), библиотека ui.buttons, ui.sidepanel, ui.notification.
Локализация: Обязательное использование языковых файлов через Loc::getMessage.
Безопасность:
Расположение строго в /local/modules/.
Проверка check_bitrix_sessid().
Проверка прав через CCrmAuthorizationHelper перед выполнением любого действия.

5. План расширения (Будущие версии)
Условия отображения: Настройка видимости кнопки в зависимости от стадии сделки или значения других полей.
Интеграция с Роботами: Возможность имитировать нажатие кнопки роботом в бизнес-процессе.
Логирование: Таблица my_bpbutton_logs для хранения истории всех нажатий с фиксацией результата выполнения.
