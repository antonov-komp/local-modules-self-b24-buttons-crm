# TASK-006: Фронтенд/UX админ‑раздела «Кнопки БП»

**Дата создания:** 2026-03-16 (UTC+3, Брест)  
**Статус:** Новая  
**Приоритет:** Средний  
**Исполнитель:** Bitrix24 Программист (Vanilla JS) + лёгкий PHP

---

## Описание

Задача описывает доработки фронтенда и UX в административном разделе модуля «Кнопки БП» поверх уже реализованного админ‑грида (`TASK-002-admin-grid`).

Цель — сделать работу администратора более удобной и «битриксообразной» за счёт:

- понятных тултипов и визуальных подсказок;
- inline‑переключения активности (`ACTIVE`) без перехода в форму;
- дружелюбных уведомлений в стиле Bitrix24 (`BX.UI.Notification`);
- (опционально) открытия форм редактирования в `BX.SidePanel` вместо традиционных «полнолистовых» страниц.

## Ступенчатые подзадачи

### 1. Тултипы и визуальные подсказки

1.1. **Тултип для `HANDLER_URL`**

- В колонке `HANDLER_URL`:
  - отображать укороченный URL (обрезка до разумной длины);
  - при наведении показывать полный URL во всплывающей подсказке:
    - либо стандартный HTML‑`title`;
    - либо через `BX.PopupWindow`/`BX.UI.Hint` (если хотите более богато).

- При реализации учитывать:
  - длинные URL не должны ломать верстку таблицы;
  - при копировании адреса администратору должно быть удобно получить полный URL.

1.2. **Подсказки по ширине (`WIDTH`)**

- Для значения `WIDTH` (пример: `70%` или `1200`):
  - добавить короткий текст‑подсказку (в виде `title` или отдельной иконки), объясняющий интерпретацию:
    - число без суффикса — пиксели;
    - строка с `%` — проценты ширины экрана/SidePanel.

1.3. **Подсказки по активности (`ACTIVE`)**

- Для поля `ACTIVE` (иконка/чекбокс) добавить:
  - локализованные подписи (например, «Включена», «Выключена»);
  - при наведении — текст «Кнопка доступна пользователям CRM / Кнопка временно отключена».

---

### 2. Inline‑переключение активности (`ACTIVE`)

2.1. **Изменение UI в списке**

- В колонке `ACTIVE`:
  - вместо/поверх статичного отображения предусмотреть:
    - либо чекбокс, либо иконку‑переключатель (например, зелёный/серый индикатор);
  - при клике по чекбоксу/иконке:
    - отправлять AJAX‑запрос на смену статуса `ACTIVE` без перезагрузки страницы.

2.2. **AJAX‑обработчик**

- Реализовать лёгкий PHP‑endpoint (или обработку внутри `bpbutton_list.php` с проверкой `$_POST['action']`), который:
  - принимает `ID` записи и новое значение `ACTIVE` (`Y`/`N`);
  - проверяет права пользователя:
    - только администратор модуля может менять статус;
  - обновляет `my_bpbutton_settings.ACTIVE` через `SettingsTable::update`;
  - возвращает JSON:

```json
{
  "success": true,
  "data": {
    "id": 123,
    "active": "Y"
  }
}
```

или при ошибке:

```json
{
  "success": false,
  "error": {
    "message": "Недостаточно прав для изменения статуса."
  }
}
```

2.3. **JS‑слой для inline‑активации**

- В админ‑списке подключить расширение `my_bpbutton.admin_list`:
  - инициализация по `BX.ready`;
  - поиск элементов‑переключателей активности (по CSS‑классу, например, `.js-bpbutton-active-toggle`);
  - навешивание обработчиков клика:
    - отправка AJAX‑запроса;
    - показ состояния «ожидание» (иконка, дизабл кнопки, и т.п.);
    - по ответу — обновление состояния и показ уведомления (см. ниже).

---

### 3. Дружелюбные уведомления Bitrix‑стилем

3.1. **Успешные операции**

- При успешном inline‑переключении `ACTIVE`:

```javascript
BX.UI.Notification.Center.notify({
    content: BX.message('MY_BPBTN_ADMIN_ACTIVE_CHANGED'), // локализованная строка
    autoHideDelay: 3000
});
```

- Текст сообщения:
  - короткий и понятный, например: «Статус кнопки обновлён.»

3.2. **Ошибки и валидация**

- При ошибках:
  - недостаточно прав;
  - не удалось сохранить изменения;
  - некорректные данные:

```javascript
BX.UI.Notification.Center.notify({
    content: errorMessage,
    autoHideDelay: 5000
});
```

- Сообщения должны быть:
  - локализованы;
  - без технических деталей (SQL, stack trace и т.п.).

3.3. **Согласованность с `style_guide`**

- Убедиться, что:
  - не используются самодельные всплывающие блоки вместо стандартных уведомлений;
  - цвета и отступы соответствуют рекомендациям `ui_ux/style_guide.md`.

---

### 4. SidePanel‑открытие формы редактирования (опционально)

4.1. **Режим открытия**

- По клику на название поля или на иконку «Редактировать»:
  - вместо перехода на отдельную страницу `bpbutton_edit.php` можно:
    - открыть её в `BX.SidePanel` с заданной шириной (например, 50–70% ширины экрана).

4.2. **Реализация**

- В списке:
  - изменить ссылку «Редактировать» или заголовок поля:
    - вместо прямого перехода по `href`:
      - навесить JS‑обработчик;
      - вызвать:

```javascript
BX.SidePanel.Instance.open(url, {
    width: '60%',
    cacheable: false,
    allowChangeHistory: true
});
```

- На стороне PHP:
  - форма редактирования (`bpbutton_edit.php`) должна корректно работать внутри SidePanel:
    - без лишней обвязки layout’а;
    - с аккуратными отступами и стандартными admin‑кнопками («Сохранить», «Применить», «Отмена»).

4.3. **UX‑ожидания**

- Администратор:
  - остаётся в контексте списка (реестр на заднем плане);
  - может быстро открыть/закрыть настройку одной кнопки, не теряя фильтры, сортировку и позицию.

## API‑методы

В рамках данной задачи новые REST‑методы Bitrix24 **не требуются**.

Допускается добавление внутренних admin‑AJAX‑endpoint’ов для:

- смены `ACTIVE` (inline‑операции);
- (опционально) вспомогательных данных для тултипов, если их нельзя собрать на клиенте.

Ответы endpoint’ов должны быть:

- в JSON‑формате с полями `success`, `data`/`error`;
- без утечки технических деталей в `error.message`.

---

## Технические требования

- Только Vanilla JS + admin‑ядро Bitrix, без Vue/React.
- JS‑код:
  - оформлен как расширение (`CJSCore::RegisterExt` + `CJSCore::Init`);
  - не создаёт глобальных переменных, кроме одного неймспейса (например, `BX.MyBpButton.AdminList`).
- Используются стандартные UI‑компоненты:
  - кнопки и элементы списка — admin‑компоненты и `ui-btn`‑классы;
  - уведомления — только через `BX.UI.Notification.Center`.
- Inline‑операции:
  - не должны ломать состояние списка (фильтры, сортировку);
  - при ошибках возвращают UI в исходное состояние.
- Все тексты (подсказки, уведомления) вынесены в языковые файлы модуля (`lang/ru/admin/*.php`) и подставляются через `Loc::getMessage`/`BX.message`.

## Критерии приёмки

- [ ] В колонке `HANDLER_URL` длинные URL отображаются аккуратно, полный текст доступен через подсказку.
- [ ] В колонке `WIDTH` есть понятная подсказка о том, как интерпретируется значение (px или %).
- [ ] В колонке `ACTIVE` реализовано inline‑переключение:
  - [ ] клик по чекбоксу/иконке меняет статус без перезагрузки страницы;
  - [ ] при успешном сохранении показывается уведомление в стиле Bitrix24;
  - [ ] при ошибке статуса (нет прав, сбой) отображается понятное сообщение и состояние UI откатывается.
- [ ] (Если включено SidePanel‑открытие) форма редактирования кнопки открывается в SidePanel, а не на отдельной странице, при этом:
  - [ ] список и фильтры сохраняют своё состояние;
  - [ ] сохранение в форме обновляет данные в списке (либо после обновления страницы, либо через JS, если предусмотрено).
- [ ] Все уведомления и подсказки локализованы и не содержат технических деталей.
- [ ] JS‑код не создаёт ошибок в консоли при обычном использовании и при типичных ошибках (нет прав, ошибка сохранения).

## Тестирование

**Функциональные проверки:**

- Открыть реестр «Кнопки БП»:
  - убедиться, что:
    - в колонке `HANDLER_URL` короткие значения, полный URL доступен в подсказке;
    - подсказки по `WIDTH` и `ACTIVE` отображаются и понятны.
- Проверить inline‑смену `ACTIVE`:
  - включить/выключить несколько записей подряд;
  - убедиться, что значения в БД (`my_bpbutton_settings.ACTIVE`) реально меняются.

**Проверка прав:**

- Под администратором модуля:
  - все описанные операции доступны и работают.
- Под пользователем без прав администратора:
  - раздел либо недоступен целиком, либо inline‑переключение `ACTIVE` не работает и показывает понятное сообщение об ошибке.

**UX‑проверки:**

- Проверить работу в светлой и тёмной темах:
  - тултипы, уведомления, иконки и кнопки выглядят корректно.
- (При использовании SidePanel):
  - открыть/закрыть форму редактирования несколько раз подряд;
  - убедиться, что список не «теряется» и не перерисовывается некорректно.

**Технические проверки:**

- В консоли браузера нет неотловленных исключений при:
  - массовом переключении `ACTIVE`;
  - ошибках сохранения;
  - открытии/закрытии SidePanel.
