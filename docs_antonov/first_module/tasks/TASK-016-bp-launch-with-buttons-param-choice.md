# TASK-016: Четвёртый формат кнопки — запуск БП с выбором значения через набор кнопок

**Дата создания:** 2026-04-22 (UTC+3, Брест)  
**Статус:** Новая  
**Приоритет:** Высокий  
**Исполнитель:** Bitrix24 Программист (коробка, D7 + Vanilla JS)

## Описание

После успешного выполнения `TASK-015` нужно добавить **новый (4-й) тип действия** кнопки:

- `bp_launch_with_button_params` (рабочее имя).

Сценарий:
1. В настройках указывается имя параметра БП (EN).
2. В настройках указывается заголовок UI для popup.
3. В настройках через кнопку «Добавить» создаётся **несколько вариантов кнопок**.
4. Пользователь в карточке CRM нажимает основную кнопку поля.
5. Открывается popup с заголовком и набором кнопок-вариантов.
6. Клик по варианту запускает БП и передаёт в параметр БП значение выбранной кнопки (её название/заголовок).

## Бизнес-правило (факт регистрации типа)

Данный документ фиксирует внедрение нового типа №4:

- Тип: `bp_launch_with_button_params`.
- Назначение: запуск БП с предопределённым набором значений через UI-кнопки.
- Факт клика по любой кнопке варианта = факт запуска БП + передача значения в параметр.

## Контекст и принцип работы

Принцип такой же, как в задаче с параметром, но без ручного ввода строки:

1. Сначала в шаблон БП передаётся **имя параметра** (EN).
2. Далее задаётся **заголовок для UI**.
3. Далее добавляется список вариантов через «Добавить» (минимум 1, обычно несколько).
4. В runtime в popup отображаются эти варианты как кнопки.
5. Выбранный вариант отправляется на backend и в `CBPDocument::StartWorkflow` уходит как:
   - `<PARAM_NAME> => <BUTTON_TITLE_OR_CODE>`.

## Тестирование

1. Создать/изменить настройку поля `bp_button_field`.
2. Выбрать тип `bp_launch_with_button_params`.
3. Заполнить:
   - `PARAM_NAME = action_result`
   - `PARAM_TITLE = Выберите действие`
   - Варианты: `Согласовано`, `На доработку`, `Отклонено`.
4. В CRM-карточке нажать кнопку поля.
5. В popup нажать, например, `На доработку`.
6. Проверить в БП:
   - старт workflow успешен;
   - параметр `action_result` получил `На доработку`.
7. Негативные кейсы:
   - пустой список вариантов;
   - дубли вариантов;
   - нет прав на запуск БП;
   - удалённый/неактивный шаблон БП.
