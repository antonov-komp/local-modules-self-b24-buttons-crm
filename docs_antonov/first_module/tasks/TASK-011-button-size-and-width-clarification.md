# TASK-011: Размер кнопки и уточнение поля «Ширина»

**Дата создания:** 2026-03-16 (UTC+3, Брест)  
**Статус:** Завершена  
**Приоритет:** Средний  
**Исполнитель:** Bitrix24 Программист (коробка, D7 + Vanilla JS)

## Описание

Задача описывает добавление настройки **размера кнопки** в карточке CRM и уточнение назначения поля **«Ширина»** в форме настроек модуля «Кнопки БП».

**Проблема:** Пользователь ожидал, что поле «Ширина» влияет на размер самой кнопки в карточке CRM. На самом деле «Ширина» задаёт только ширину боковой панели (SidePanel), которая открывается при клике. Размер кнопки настраивался не был.

**Решение:** Добавлено отдельное поле «Размер кнопки» с вариантами: стандартная, маленькая, большая. Поле «Ширина» переименовано в «Ширина панели» с уточнённой подсказкой.

## Контекст

- Форма настроек кнопки — `admin/bpbutton_list.php`.
- Рендеринг кнопки — `BpButtonUserType::getPublicViewHTML`.
- Таблица настроек — `my_bpbutton_settings`.

## Модули и компоненты

- `local/modules/my.bpbutton/lib/Internals/SettingsTable.php` — поле `BUTTON_SIZE`.
- `local/modules/my.bpbutton/install/index.php` — миграция колонки `BUTTON_SIZE`.
- `local/modules/my.bpbutton/lib/EventHandler.php` — `ensureButtonSizeColumnExists()`, добавление `BUTTON_SIZE` в `onAfterUserFieldAdd`.
- `local/modules/my.bpbutton/admin/bpbutton_list.php` — поле формы «Размер кнопки», сохранение `BUTTON_SIZE`.
- `local/modules/my.bpbutton/lib/UserField/BpButtonUserType.php` — `getButtonSizeForField()`, применение inline-стилей к кнопке.
- `local/modules/my.bpbutton/lang/ru/admin/bpbutton_list.php` — локализация.

## Реализованные подзадачи

### 1. Добавление поля BUTTON_SIZE в SettingsTable ✅

- `SettingsTable::getMap()` — `Entity\StringField('BUTTON_SIZE', ['validation' => Length(null, 20)])`.
- Миграция в `install/index.php`: колонка `BUTTON_SIZE VARCHAR(20) NULL AFTER WIDTH`.
- `EventHandler::ensureButtonSizeColumnExists()` — для существующих установок.

### 2. Форма редактирования ✅

- Добавлен выпадающий список «Размер кнопки» с вариантами: `default`, `sm`, `lg`.
- Локализация: `MY_BPBUTTON_EDIT_FIELD_BUTTON_SIZE`, `_HINT`, `_DEFAULT`, `_SM`, `_LG`.
- Сохранение в AJAX- и обычном POST-обработчиках.

### 3. Уточнение поля «Ширина» ✅

- Переименовано в «Ширина панели».
- Подсказка: «Ширина боковой панели (SidePanel), которая откроется при клике. Пиксели (800) или проценты (80%). Не влияет на размер самой кнопки.»

### 4. Применение размера к кнопке ✅

- `BpButtonUserType::getButtonSizeForField()` — загрузка из `SettingsTable` с кешированием по `FIELD_ID`.
- Варианты:
  - `default` — без дополнительных стилей;
  - `sm` — `padding: 4px 12px; font-size: 12px;`;
  - `lg` — `padding: 12px 28px; font-size: 16px;`.
- Inline-стили добавляются к атрибуту `style` кнопки в `getPublicViewHTML`.

### 5. EventHandler и авто-синхронизация ✅

- `onAfterUserFieldAdd` — `BUTTON_SIZE` => null при создании настроек.
- Авто-синхронизация в списке (`bpbutton_list.php`) — `BUTTON_SIZE` => null при создании записей для существующих полей.

## Критерии приёмки

- [x] В форме настроек кнопки отображается поле «Размер кнопки» с вариантами: стандартная, маленькая, большая.
- [x] При выборе «Большая» кнопка в карточке CRM визуально увеличивается.
- [x] При выборе «Маленькая» кнопка визуально уменьшается.
- [x] Поле «Ширина панели» имеет подсказку, что оно не влияет на размер кнопки.
- [x] Миграция добавляет колонку `BUTTON_SIZE` для существующих установок.

## Тестирование

- **Сценарий 1:** Админка → Кнопки БП → Редактировать → выбрать «Размер кнопки» = «Большая» → сохранить. Открыть карточку CRM с полем. Ожидание: кнопка крупнее стандартной.
- **Сценарий 2:** Выбрать «Маленькая» → сохранить. Ожидание: кнопка меньше стандартной.
- **Сценарий 3:** Проверить подсказку у «Ширина панели» — указано, что это ширина SidePanel, не кнопки.
